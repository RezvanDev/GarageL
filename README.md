Документация проекта Garage 🚗💨
Этот документ содержит описание архитектуры, технологий и инструкций по работе с проектом Garage (Клиентское Mini App + Бэкенд).

🚀 Технологический стек
Бэкенд (backend/)
Node.js: Среда выполнения.
Express 5: Веб-фреймворк для API.
PostgreSQL: Основная база данных.
JWT (JSON Web Token): Аутентификация.
bcryptjs: Хеширование паролей.
cors: Настройка доступа для фронтенда.
Фронтенд (frontend/)
React 19: Библиотека интерфейсов.
Vite: Сборщик (очень быстрый).
Framer Motion: Премиальные анимации.
Lucide React: Иконки.
Telegram Web App SDK: Интеграция с Telegram.
🏗️ База данных и ORM
На текущем этапе я не использовал Prisma или Drizzle. Вместо этого я применил чистый SQL через драйвер pg (node-postgres).

Почему не Prisma?
Скорость и контроль: Чистый SQL позволяет писать максимально эффективные запросы.
Простота: Для текущей схемы данных установка сложного ORM была бы избыточной.
Прозрачность: Вы точно видите, какие таблицы создаются в файле 
backend/src/db/schema.sql
.
Как работать с БД:
Конфигурация: Находится в 
backend/src/db/index.js
.
Схема: Полная структура таблиц лежит в 
backend/src/db/schema.sql
.
Модели: Запросы к БД разделены по файлам в backend/src/models/ (User, Product, Order).
Сид (Тестовые данные): Чтобы наполнить базу товарами, запустите node src/db/seed.js.
📁 Структура проекта
Бэкенд
/src/controllers: Логика обработки запросов (что делать).
/src/routes: Маршруты API (куда заходить).
/src/models: SQL-запросы к базе данных.
/src/middleware: Проверки (например, авторизован ли пользователь).
/src/db: Настройка подключения и схемы.
Фронтенд
/src/views: Основные экраны (Dashboard, Catalog, Auth).
/src/components: Повторяющиеся элементы (Кнопки, Карточки).
/src/hooks: Логика состояния (useGarageState.js).
/src/services: Связь с бэкендом (api.js).
🛠️ Как запустить и работать
1. Настройка Бэкенда
Перейдите в backend/.
Установите зависимости: npm install.
Настройте .env (убедитесь, что DATABASE_URL верный).
Создайте таблицы: скопируйте содержимое src/db/schema.sql в вашу консоль PostgreSQL (или используйте psql).
Запустите: npm run dev. (Сервер поднимется на порту 5001).
2. Настройка Фронтенда
Перейдите в frontend/.
Установите зависимости: npm install.
Запустите: npm run dev.
Откройте http://localhost:5173.
🛡️ Безопасность (RBAC)
В системе уже заложена ролевая модель:

client: Доступ только к своим заказам и каталогу.
supplier: Будет видеть запросы клиентов.
logistician: Будет управлять доставкой.
admin: Полный доступ.
Проверка ролей происходит на бэкенде через middleware restrictTo('admin', ...).
